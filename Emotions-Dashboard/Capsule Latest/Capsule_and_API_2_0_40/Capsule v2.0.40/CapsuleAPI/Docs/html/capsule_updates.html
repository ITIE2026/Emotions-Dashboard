Capsule Version Updates

Capsule v 1.5.0

We dedicated this release to many small improvements - both in terms of the Capsule itself and the API.

The Capsule utility itself and the API (with the exception of the raw signal from the sensors) require entering a license key to work.

Capsule

  • Improved calculation of cardiometrics (PPG)
  • Improved work with devices when searching and initializing a connection frequently.
  • Ability to change the averaging period for recording metrics in XLSX in the interface (5, 10, 15, 30 seconds)
  • Reworked data sources for the emotional state classifier
  • The mechanism for sending data to the platform has been redesigned, sending queues have been separated
  • Fixed incorrect behavior when opening a DMG package on MacOS versions lower than 13

API

  • Expanded error logging when trying to connect a device without pairing for iOS and MacOS

Capsule v 1.4.5

API

  • Fixed a bug when forcibly disconnecting a device in Android OS

Capsule v 1.4.0

Starting from this version, in order to use ready-made PSBD metrics in the API and the Capsule graphical utility, you will need to purchase and enter a license key. To work with raw streaming signal, a license key is not required.

At the first launch, Internet access will be required for testing; for subsequent launches, there is a period when access may not be available.

Capsule

  • The configuration of digital high-pass filters has been changed, the average number of artifacts per channel has been reduced by 23%
  • Accumulated fatigue growth rate in GUI (Low, Medium, High)
  • Graph of changes in detected psychophysiological states over time
  • Indication of PPG artifacts
  • Version for Android OS family

API

  • Changed methods clCClient_CreateSession, clCClient_CreateSessionWithMonopolarChannels New method names: clCClient_CreateSessionWithError, clCClient_CreateSessionWithMonopolarChannelsWithError
  • The clCNFBMetricsProductivityValues ​​structure has been expanded and the fatigueGrowthRate parameter has been added
  • Added clCLicenseManager class for license checking and methods for working with it
  • Added method clCClient_GetLicenseManager - initialize the license manager
  • Added method clCLicenseManager_VerifyLicense - send a license verification request
  • Added method clCLicenseManager_IsLicenseValid - check license status
  • API documentation updated
